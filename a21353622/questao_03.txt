working directory é o que o git chama de area de trabalho serve para alocar os arquivos não monitorados que estão dentro da pastas em que foi dada o git init

index é o que o git chama área de monitoramento ou seja, depois que você executa o comando "git add" os seus arquivos vai para uma área monitorada para que em seguida o commit grave as mudanças.

repository é o que o git chama de área de persistência, depois que você executa o comando "git commit" suas modificações ficam salvas no seu git e a partir desse momento é possível recuperar arquivos a partir o cheksum que é gerado.
